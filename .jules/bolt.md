## 2024-04-30 - Fix N+1 queries on DRF bulk_create
**Learning:** By default, when a Django Rest Framework (DRF) serializer with `many=True` handles bulk creation (e.g. `ContactosSerializer(data=..., many=True)`), the internal `ListSerializer.create` falls back to looping over each item and calling the child serializer's `create` method. For `ContactosSerializer`, this meant repeatedly executing `Contacto.objects.update_or_create(...)` which generated multiple sequential queries (N+1 bottleneck) and heavily slowed down batch processing in large data operations.
**Action:** Always verify if a model serializer that commonly processes large arrays provides a custom `ListSerializer`. Implement and assign a custom `ListSerializer` subclass via the child's `Meta.list_serializer_class` and override its `create` method to utilize PostgreSQL's native `bulk_create` (along with `update_conflicts=True` to replicate `update_or_create` semantics) to optimize away the repetitive database roundtrips.
